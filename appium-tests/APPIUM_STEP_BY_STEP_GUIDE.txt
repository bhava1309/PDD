1. What This Testing Folder Contains

- Appium Java test project using Maven and TestNG.
- End-to-end Android mobile tests for the PawPal app.
- More than 400 total test executions through fixed E2E flows and data-driven validation tests.
- Excel report generation using Apache POI.
- Excel report format matching the provided E2E reference workbook:
  Summary, Passed Tests, Failed Tests, Execution Log, Test Details.
- Failure screenshot capture.
- Appium server runner script.

2. Main Files

- pom.xml
  Defines Appium, Selenium, TestNG, Apache POI and reporting dependencies.

- run_tests.sh
  Starts the Appium run from one command.

- src/test/resources/testng.xml
  Controls the complete E2E test order.

- src/test/java/com/pawpal/tests/base/BaseTest.java
  Starts the Android Appium driver and switches into the PawPal WebView.

- src/test/java/com/pawpal/tests/config/AppConfig.java
  Stores APK path, device name, app package, app activity and test data.

- src/test/java/com/pawpal/tests/utils/ExcelReporter.java
  Creates the Excel analysis report after the test run.

- src/test/java/com/pawpal/tests/tests/
  Contains all PawPal E2E test classes.

4. One-Time Setup

Install these tools on the machine:

- Java 11 or newer
- Maven
- Node.js
- Appium 2
- Android Studio
- Android SDK platform tools
- Appium UiAutomator2 driver

Install Appium driver once:

appium driver install uiautomator2

5. Start Android Device

Use any one option:

- Start an Android emulator from Android Studio Device Manager.
- Or connect a real Android phone with USB debugging enabled.

Check device connection:

adb devices

6. Run Complete Appium E2E Test

Open terminal in:

/Users/surya/Documents/pawpal/appium-tests

Run:

./run_tests.sh

Run with a custom APK:

./run_tests.sh --apk "/Users/surya/Documents/pawpal/PawPal-Android-Install.apk"

Run with a custom device:

./run_tests.sh --device "emulator-5554"

Run with another Appium server port:

./run_tests.sh --server "http://127.0.0.1:4725"
